为了方便各位用户及开发者更容易调阅 FimTale 上的信息，一部分页面被以 JSON 字符串的形式暴露在 API 当中。这便是 FimTale 的第一版 API。

**为缓解站点访问压力，API 仅发放给已注册且具备 API 权限的用户。每名用户可在设置页面中管理不多于 3 对 API 登录信息（`APIKey` 为 8 位、`APIPass` 为 12 位，均为小写十六进制字符串）。在提交请求时，请将 `APIKey` 与 `APIPass` 一并附上。请不要过于频繁地使用 API 访问本站；对于影响到站点功能的访问，我们将考虑禁用该用户继续使用 API 的权限。**

### API 的使用方法

在网址的域名（如 `https://fimtale.com`）与路径（`/` 及后面的部分）之间加入 `/api/v1` 即可。

例如：

- `https://fimtale.com/t/4` — 作品页面；
- `https://fimtale.com/api/v1/t/4?APIKey={您的 APIKey}&APIPass={您的 APIPass}` — 对应作品的第一版 API。

下文示例 URL 均省略 `APIKey` 与 `APIPass`，实际请求时必须携带。

#### 认证

| 项目 | 说明 |
| --- | --- |
| 参数名 | `APIKey`、`APIPass` |
| 传递方式 | 可通过 **GET 查询字符串** 或 **POST 表单字段** 传递（二者等效，服务端从 `$_REQUEST` 读取） |
| 校验逻辑 | 在 `carbon_usermeta` 中匹配 `api_key` / `api_pass` 配对，且对应账号状态正常并具备 `APIPermission` |
| 失败响应 | `Status` 为 `0`，`ErrorCode` 通常为 `403`，`ErrorMessage` 为错误说明 |
| CORS | 响应头包含 `Access-Control-Allow-Origin: *` |
| 响应格式 | `Content-Type: application/json; charset=utf-8` |

**重要：通过校验的 `APIKey` / `APIPass` 会同时作为本次 API 请求的用户身份。** 响应中的 `CurrentUser`，以及各对象上的 `IsFavorite`、`MyVote`、`AllowedOptions` 等字段，均以**密钥所属账号**为准。`/api/v1` 请求会**忽略**浏览器中的站点登录 Cookie，避免「携带他人密钥、却套用本机浏览器会话」造成的身份错配。

因此，调用 `getMyInfo`、`/settings` 等需要登录的接口时，**只需**提供该用户自己的有效 `APIKey` 与 `APIPass`，无需 additionally 维持网页登录 Cookie。旧版客户端继续通过查询参数或表单提交这两个字段即可，参数名与格式不变。

在开发模式（`DEVELOP_MODE`）下，若未配置密钥校验，则回退为使用 Cookie 中的 `UserID` 作为 API 身份。

#### 内容格式

API 中作品、便签、评论等正文字段默认输出为 **HTML**。若需要 Markdown，可在查询参数中加上 `format=md`。

#### FormHash

通过 API 访问时，服务端会跳过 `FormHash` 校验（`ReferCheck` 在 API 请求下恒为通过）。但 `CurrentUser.FormHash` 仍会在已登录时返回，供需要写操作的网页表单使用。

### 补充内容（`/json/{action}`）

路径中的 `{action}` 为操作名，例如 `/api/v1/json/getInfoByURL`。以下接口均仍需携带 API 密钥；标注「需登录」者 additionally 需要会话 Cookie。

#### `getInfoByURL` — 通过 URL 批量获取卡片信息

| 参数 | 类型 | 描述 |
| --- | --- | --- |
| `urls` | `String` | JSON 编码的 URL 字符串列表 |

| 返回值 | 描述 |
| --- | --- |
| `BlogpostArray` | 便签信息 |
| `ChannelArray` | 频道信息 |
| `TagArray` | 标签信息 |
| `TopicArray` | 作品信息 |
| `UserArray` | 用户信息 |
| `RequestList` | 解析后的 ID 分组 |

示例：`https://fimtale.com/api/v1/json/getInfoByURL?urls=%5B%22https%3A%2F%2Ffimtale.com%2Ft%2F4%22%5D`

#### `getTags` — 获取可选标签列表（需登录）

### 设置（`/settings`）API

GET 需登录，返回当前设置状态；POST 需登录，以 `Action` 指定操作。API 请求无需 `FormHash`。

##### 只读结构（节选）

| 键名 | 描述 |
| --- | --- |
| `Messages` | 各 `Action` 最近一次操作的消息 |
| `Console` | 控制台相关；含 `Badges.Displayed` / `Badges.Hidden` |
| `UploadAvatar` | 头像上传表单占位 |
| `UpdateUserInfo` | 用户资料当前值 |
| `SetGrandFilter` | 大滤镜开关 |
| `BlackLists` | 已屏蔽用户（`Users`）与标签（`Tags`） |

`UpdateUserInfo` 中还包含 `UserMailVerified`、`MutedNotificationReplyRange`、`MutedNotificationMentionRange`、`MutedNotificationInboxRange`、`MutedNotificationInteractionRange` 等，取值可为 `"all"`、`"favonly"`、`"none"`、`"nocommentupvote"`。

`SetGrandFilter` 键名为 `NoRestricted`、`NoRedTags`（`0` / `1`）。

### 安全提示

- 切勿在公开仓库、前端 code 或日志中明文保存 `APIPass`（访问日志已对 `APIKey` / `APIPass` 脱敏）。
- `APIKey` / `APIPass` 出现在 URL 查询字符串中时，可能被代理、浏览器历史或 Referer 泄露；建议在服务端调用时优先使用 POST 传递。
- 在第三方页面中调用 API 时，请勿同时携带其他用户的站点登录 Cookie；服务端会以密钥所属账号为准，Cookie 不会覆盖 API 身份。
- 泄露后请在设置页删除对应密钥并重新生成。
